<objective>
Corrigir bug destrutivo na função `resolve_worktree` que apagava branches locais existentes silenciosamente quando `git worktree add` falhava por qualquer razão.

**Purpose:** O launcher hoje pode destruir trabalho do usuário sem aviso. Tentar criar worktree de uma branch que já existe localmente (caso comum de "puxei origin/foo, agora quero trabalhar em foo") falha e dispara `git branch -D foo`, deletando a branch local que poderia conter commits.

**Output:** `resolve_worktree` modificada para usar snapshot before/after — só deleta branches que **nasceram nesta invocação**. Em qualquer ambiguidade, não deleta nada.
</objective>

## T1.3 — Remover cleanup destrutivo cego, substituir por cleanup condicional

**Onde:** `repo-launcher.sh:473-484` (bloco "Cleanup: if the branch was created but worktree failed").

**Antes (atual, destrutivo):**

```bash
case "$type" in
  new)
    (cd "$repo_path" && git branch -D "$branch" >/dev/null 2>&1) || true
    ;;
  remote)
    local local_name="${branch#*/}"
    (cd "$repo_path" && git branch -D "$local_name" >/dev/null 2>&1) || true
    ;;
esac
rmdir "$wt_path" 2>/dev/null || true
```

**Depois (snapshot-based, seguro):**

```bash
# Determinar nome da branch que SERIA criada
local target_branch=""
case "$type" in
  new)    target_branch="$branch" ;;
  remote) target_branch="${branch#*/}" ;;
  # local: nenhuma branch é criada, só o worktree — sem cleanup de branch
esac

# Cleanup só se: a branch alvo existe AGORA mas NÃO existia antes (=nasceu nesta call)
if [[ -n "$target_branch" ]]; then
  local post_branches
  post_branches="$( (cd "$repo_path" && git branch --format='%(refname:short)' 2>/dev/null) | LC_ALL=C sort )"
  if printf '%s\n' "$post_branches" | grep -qx -- "$target_branch" \
     && ! printf '%s\n' "$pre_branches" | grep -qx -- "$target_branch"; then
    # Nasceu agora E falhou → órfã, pode deletar com segurança
    (cd "$repo_path" && git branch -D "$target_branch" >/dev/null 2>&1) || true
  fi
  # Se já existia antes (case do bug), NÃO toca.
fi

# Cleanup do dir órfão é seguro (não destrutivo)
rmdir "$wt_path" 2>/dev/null || true
```

**Por que `grep -qx --`:**
- `-x` exige match da linha inteira (não substring)
- `-q` silencia output
- `--` evita interpretar nome de branch começando com `-` como flag

**Validação:** code review visual + smoke test (T1.4).
